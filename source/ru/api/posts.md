## Создание поста

``` js
hexo.post.create(данные, замену);
```

| Аргумент   | Описание                    |
| ---------- | --------------------------- |
| `данные`   | Данные                      |
| `заменить` | Заменять существующие файлы |

Атрибуты поста можно установить в переменной `data`. Таблица ниже не является полной. Дополнительные атрибуты могут быть добавлены в шапке поста.

| Данные     | Описание                                                                                          |
| ---------- | ------------------------------------------------------------------------------------------------- |
| `название` | Заголовок                                                                                         |
| `слаг`     | Ссылка                                                                                            |
| `макет`    | Шаблон. По умолчанию в настройках указано `default_layout`.                                       |
| `путь`     | Путь. По умолчанию Hexo строит пути на основе переменной `new_post_path`, указанной в настройках. |
| `дата`     | Дата. По умолчанию — текущая дата.                                                                |

## Опубликовать черновик

``` js
hexo.post.publish(данные, замена);
```

| Аргумент   | Описание                    |
| ---------- | --------------------------- |
| `данные`   | Дата                        |
| `заменить` | Заменять существующие файлы |

Атрибуты поста можно установить в переменной `data`. Таблица ниже не является полной. Дополнительные атрибуты могут быть добавлены в шапке поста.

| Данные  | Описание                                                                             |
| ------- | ------------------------------------------------------------------------------------ |
| `слаг`  | Имя файла (Обязательно)                                                              |
| `макет` | Шаблон. По умолчанию берется из переменной `default_layout`, указанной в настройках. |

## Рендер

``` js
hexo.post.render(источник, данные);
```

| Аргумент   | Описание                            |
| ---------- | ----------------------------------- |
| `источник` | Полный путь к файлу (необязательно) |
| `данные`   | Данные                              |

Данные должны содержать атрибут `content`. Если нет, Hexo постарается прочитать исходный файл. Этапы выполнения этой функции следующие:

- Предобработка фильтрами `before_post_render`
- Обработка с помощью Markdown или другого обработчика (выбирается в зависимости от расширения)
- Обработка [Nunjucks][]
- Постобработка фильтрами `after_post_render`

[Nunjucks]: https://mozilla.github.io/nunjucks/
